# DBMS ERD Assignments

This repository contains a collection of Entity-Relationship (ER) diagrams designed for real-world systems as part of DBMS practice and coursework.

## Objective

The goal of this repository is to strengthen database design skills by modeling practical applications with clean, scalable, and well-structured schemas.

## Concepts Covered

- Entity and relationship identification  
- One-to-Many and Many-to-Many relationships  
- Use of junction tables  
- Primary and Foreign Keys  
- Normalization techniques  
- Real-world system modeling  

## Repository Structure

Each assignment is organized in its own folder and includes:

- `erd.png` → Visual representation of the ER diagram  
- `schema.dbml` → Source file used to generate the diagram  

## Assignments Included

- Instagram Thrift Store Database  
- Fitness Influencer Coaching Platform Database  
- *(More assignments will be added regularly)*  

## Tools Used

- eraser.io  

## Approach

Each ER diagram is created by following a structured process:

1. Understanding the business problem  
2. Identifying core entities and their relationships  
3. Designing schemas that reflect real-world workflows  
4. Ensuring proper normalization and scalability  

## Note

This repository focuses purely on database design (ER diagrams).  
No frontend or backend implementation is included.
